# ShareIt 🔄 — Сервис для шеринга вещей

**ShareIt** — это современная платформа, которая позволяет пользователям делиться своими вещами. Проект реализует полный цикл аренды: от добавления предмета в каталог до управления запросами на бронирование и возможности оставить отзыв после использования.

Приложение построено на микросервисной архитектуре, что обеспечивает масштабируемость и независимое развитие компонентов.

---

### 🏗️ Архитектура системы

Проект разделен на два ключевых сервиса:

1.  **Gateway (Шлюз):**
    * Точка входа для всех клиентских запросов.
    * Выполняет первичную валидацию данных.
    * Перенаправляет запросы в основной сервис.
2.  **Server (Основной сервис):**
    * Обрабатывает всю бизнес-логику (бронирование, работа с вещами, пользователи).
    * Взаимодействует с базой данных PostgreSQL.
    * Реализует сложную логику проверки пересечения дат бронирования.

---

### 🛠️ Технологический стек

* **Java & Spring:** Java 11+, Spring Boot, Spring Data JPA.
* **Database:** PostgreSQL, Hibernate, Querydsl (для сложных поисковых запросов).
* **Validation:** JSR-303 (Bean Validation).
* **Mapping:** MapStruct (эффективное преобразование DTO в сущности).
* **Infrastructure:** Docker & Docker Compose.
* **Testing:** JUnit, Mockito, интеграционное тестирование.

---

### 🚀 Основной функционал

* **Управление вещами:** Создание, редактирование, поиск по названию и описанию.
* **Бронирование:** Система статусов (WAITING, APPROVED, REJECTED, CANCELED) и автоматическая проверка доступности вещи на выбранные даты.
* **Запросы (Requests):** Возможность создать запрос на вещь, которой ещё нет в системе.
* **Отзывы:** Комментарии от пользователей, которые уже брали вещь в аренду.
* **Пагинация:** Все списки возвращаются постранично для оптимизации нагрузки.

---

### 📊 Схема данных

Проект использует реляционную схему, оптимизированную для частых поисковых запросов и связей «многие-к-одному» между бронированиями, пользователями и вещами. Интеграция с **PostgreSQL** обеспечивает транзакционность и целостность данных.
